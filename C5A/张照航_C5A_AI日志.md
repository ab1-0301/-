# C5A 挑战 AI 使用日志

## 使用场景 1：理解仓库分析文档的结构
- **我的提问**：“仓库分析文档需要包含哪几个部分？能不能给个模板？”
- **AI 的回答**：给出了四个问题（仓库用途、目录结构、技能列表、最喜欢的技能）以及 Markdown 表格示例。
- **我的行动**：按照模板填写了 `张照航_C5A_仓库分析.md`，并补充了从 `src/` 目录观察到的结构。

## 使用场景 2：解决 Commit 按钮灰色的问题
- **我的提问**：“Commit 按钮是灰色的，怎么办？”
- **AI 的回答**：指出需要填写提交信息（标题），并附上截图示意。
- **我的行动**：在标题框输入了“添加中文学习说明”，按钮变绿，成功提交。

## 使用场景 3：找不到 commit 历史
- **我的提问**：“如何找到我的 commit 记录？”
- **AI 的回答**：提供了两种方法：点击分支下拉菜单中的 `commits` 链接，或直接访问 `https://github.com/用户名/neoskills/commits/my-contribution`。
- **我的行动**：通过该 URL 找到了自己的提交并截图。

## 使用场景 4：不知道如何创建文件夹
- **我的提问**：“如何在 GitHub 网页上创建文件夹？”
- **AI 的回答**：介绍了在创建文件时通过路径名（如 `文件夹名/文件名`）自动生成文件夹的方法。
- **我的行动**：成功创建了 `张照航_C5A_screenshots/` 文件夹并上传了截图。

## 使用场景 5：撰写 AAR 和 AI 日志
- **我的提问**：“AAR 和 AI 日志怎么写？”
- **AI 的回答**：给出了两个模板，并说明了每个部分的作用。
- **我的行动**：按照模板填写了自己的真实经历。

## 总结
本次挑战中，AI 帮助我：
- 快速理解了 GitHub 操作流程
- 解决了网页操作中的具体卡点
- 提供了文档写作框架
- 节省了大量查阅资料的时间

我学会了如何更有效地向 AI 提问，例如提供截图、描述错误现象、明确请求格式。
