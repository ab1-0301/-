## 对 SKILL.md 的理解（技能工作原理）

阅读了 `pdf/SKILL.md` 后，我对 PDF 技能的理解如下：

### 技能的核心机制

PDF 技能本质上是一个 **操作指南型技能**，它不像传统软件插件那样提供 GUI 按钮，而是通过 **SKILL.md 文件** 向 AI 智能体传授处理 PDF 的专业知识和方法。

### 技能的工作原理（四层结构）

1. **触发层** — 技能头部（Frontmatter）
   - `name: pdf` — 技能的唯一标识
   - `description` — 技能触发条件描述。当用户提到 PDF 或 .pdf 文件时，系统自动匹配此技能
   - 这相当于"配方卡片"的名称和标签

2. **知识层** — 概览与快速入门
   - 提供 `pypdf`、`pdfplumber`、`reportlab` 等核心 Python 库的代码示例
   - 让 AI 了解可使用的工具生态

3. **方法层** — 分场景的操作指南
   - 文本提取 → pdfplumber
   - 表格提取 → pdfplumber + pandas
   - PDF 创建 → reportlab
   - PDF 合并/拆分 → pypdf
   - 扫描件 OCR → pytesseract + pdf2image
   - 加密/解密 → pypdf
   - 每种操作都配有 **可直接使用的代码片段**

4. **参考层** — 快速对照表与高级参考
   - 任务→工具→代码的快速映射
   - 指引到 REFERENCE.md 和 FORMS.md 获取更详细的信息

### 技能与传统程序的区别

| 特性 | 传统程序/插件 | AI 技能 (SKILL.md) |
|------|-------------|-------------------|
| 执行方式 | 用户点击按钮 | AI 读取后自主执行 |
| 知识载体 | 二进制代码 | 自然语言文档 |
| 扩展性 | 需重新编译 | 修改 MD 文件即可 |
| 组合方式 | API 调用 | 自然语言描述组合 |
| 环境依赖 | 特定运行时 | AI 理解后选择合适工具 |

### 为什么技能是"配方卡片"？

正如课程大纲 D5 所述：**"当任务匹配时从架子上取下的配方卡片。名称+描述作为触发条件；正文告诉代理该做什么。可组合。"**

PDF 技能正是这一理念的体现——它不硬编码任何 PDF 处理逻辑，而是通过精心编写的 Markdown 文档，让 AI 智能体理解：
- **什么时候**应该使用 PDF 处理能力（由 frontmatter 的 description 触发）
- **用什么工具**处理（pypdf/pdfplumber/reportlab 等）
- **什么场景用什么方法**（文本提取 vs 表格提取 vs 创建 PDF）
- **遇到特殊情况怎么办**（OCR 扫描件、Unicode 下标字符兼容性等坑点）

这正是 AI 技能的核心价值：**"在不编写代码的情况下教会智能体新技能。"**
