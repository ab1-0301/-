# K-S-T-A-R 框架

KSTAR 是 AI+X Elite20 的核心工作流框架：**Knowledge → Situation → Task → Action → Reflection**。

---

## K — 知识库（我已知的信息）

列出完成任务前已经掌握的知识、可用资料和前提条件：

- 描述已有信息（文档、数据、代码等）
- 明确自己的能力和限制
- 标记不确定或需要验证的部分

> **要点：** 诚实面对已知和未知，避免遗漏关键上下文。

---

## S — 情境（当前所处状态）

记录完成任务时的环境和约束：

| 维度 | 描述 |
|------|------|
| **地点** | |
| **设备/工具** | |
| **时间限制** | |
| **情绪/状态** | |
| **可用资源** | |

> **要点：** 情境决定了策略选择，不同情境下"好"的标准不同。

---

## T — 任务（要做什么 + 验收标准）

### 任务指令

清晰描述要完成的内容。

### 验收标准

- [ ] 标准 1（可衡量、可检查）
- [ ] 标准 2
- [ ] 标准 3

> **要点：** 验收标准必须可检查——没有验收标准的任务无法确认完成。

---

## A — 行动（执行 + 迭代）

### 迭代 1：初次尝试

**输入指令：** ...
**输出结果：** ...
**问题：** 发现了什么不足？

### 迭代 2：修正改进

**输入指令：** 补充/修正了什么
**输出结果：** 改进后的结果
**问题：** 还有没有遗留问题？

> （按需重复迭代，直到满足验收标准）

### 最终提示词（完整版）

> 经过迭代后最终使用的完整指令。

---

## R — 复盘反思（学到了什么）

- 收获 1：框架/方法论层面的体会
- 收获 2：具体技巧层面的发现
- 收获 3：下次可以改进的地方

> **要点：** 反思是 KSTAR 的闭环——没有反思就没有成长。
