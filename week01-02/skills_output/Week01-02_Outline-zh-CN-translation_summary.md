## 核心框架：AI+X Elite20 课程设计

这份文档是一份为期两周（10天）的集训课程大纲，面向混合专业背景的本科生，目标是让零基础学生从第1天到第10天完成从工具搭建到独立交付作品的完整过程。

---

### 第1页核心内容：逆向设计目标

课程采用 **逆向设计** 方法——先确定第10天结束时学生的能力：

1. **环境搭建**: 个人笔记本上运行 Claude 桌面端 / Cowork + 工作区文件夹 + 终端
2. **GitHub 使用**: 拥有 `elite20-starter` 的个人分支，提交≥3次代码
3. **技能调用**: 至少调用过一项预设技能并阅读过 SKILL.md
4. **K-S-T-A-R 循环**: 针对实际任务完整执行全流程
5. **作品交付**: 完成一项小型作品（海报/翻译/摘要/脚本等）
6. **自我定位**: 在 AI+X 3D 框架中提交一页"坐标卡"
7. **核心指标**: 能用 K-S-T-A-R 词汇在 3 分钟内阐述自己的作品

---

### 第2页核心内容：五轨并行教学法

课程采用 **多轨并行模式**（非传统线性教学），五条轨道每日交织：

| 轨道 | 主题 | 类比 |
|------|------|------|
| **T1 · 世界观** | 为何 AI+X 至关重要；3D 框架 | "正在绘制的地图" |
| **T2 · 环境配置** | 账户、笔记本、Claude、协作平台 | "工作坊" |
| **T3 · GitHub** | 版本控制 + 同行评审 | "实验记录本" |
| **T4 · 技能** | 调用与阅读 SKILL.md | "智能体读取的配方卡片" |
| **T5 · 氛围编程** | 提示→迭代→验证→交付 (K-S-T-A-R) | "爵士乐，而非视奏" |

**十日规划概览**（D1-D10）：
- 第一周（D1-D5）：环境搭建 —— 从零基础到准备就绪
  - D1: 世界观 + 账户设置
  - D2: 代理环境搭建
  - D3: GitHub 首次提交
  - D4: 提示词基础
  - **D5: 技能配方卡（调用技能 + 阅读 SKILL.md）**
- 第二周（D6-D10）：首次氛围编程循环
  - D6: 氛围编程演示
  - D7: K-S-T-A-R 拆解
  - D8: AI+X 3D 框架
  - D9: 首个氛围编程项目
  - D10: 展示与回顾

### 第3页核心内容：第一周每日详解（D1-D5）

**D1（周一）— 欢迎仪式、世界观、账户设置**
- 引导问题："如果 AI 能完成你 40% 的学习工作，你会交出哪 40%？"
- 开设 GitHub、Claude、Cowork、微信群等所有账号
- 交付：三行反思记录

**D2（周二）— 你的工作坊：智能体环境**
- 关键洞察：同一提示词在不同环境输出不同 → **环境本身就是智能体的一部分**
- 工作坊类比：工作台（文件夹）+ 墙上工具（MCP）+ 配方手册（技能）+ 学徒（智能体）
- 交付：`hello.md` 冒烟测试截图

**D3（周三）— 你的实验笔记本：GitHub**
- 四个类比：代码库=笔记本，提交=带日期页，分支=假设场景，PR=同行评审
- 操作：fork → clone → 提交 → 推送
- 交付：GitHub 上首个提交记录

**D4（周四）— 与智能体对话：提示基础**
- 清晰提示词的四个要素：角色+受众、输入内容、输出形式、验收标准
- 交付：三个提示追踪文件

**D5（周五）— 技能配方卡（★★★★★ 本任务所在）**
- 技能 = **配方卡片**：名称+描述=触发条件，正文=告诉智能体做什么
- 选择与专业相关的预构建技能调用（pdf/docx/canvas-design 等）
- 交付：**技能生成的成果物 + 一段说明**（即本任务正在完成的内容）

## 关键洞察总结

1. **"环境是智能体的一部分"** — 同一 prompt 在不同环境输出不同
2. **"爵士乐，而非视奏"** — 氛围编程是即兴交互而非机械执行
3. **"技能是配方卡片"** — SKILL.md 让 AI 在不编写代码的情况下学会新能力
4. **"对角线才是关键"** — AI+X 3D 框架的核心是 X（领域）× AI（工具）× S（自我反思）三条轴线共同移动
5. **评估标准**：第1-2周只评估能力完成度（通过/不通过），不评质量分数
